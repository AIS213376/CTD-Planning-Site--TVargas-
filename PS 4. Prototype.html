Planning Site

My wireframes for the planning site will then be used to create a storyboard. Since my website has a consistent look all throughout, I decided to create my storyboard from Dreamweaver.

In Dreamweaver, I can create a template that will be used for every webpage in my planning website.

All of the boxes in the template are AP Divs, which are divisors with absolute positioning in the file.

Everything but the two boxes next to the navigation are permanent, which means it can only be edited in the template. The rest are editable regions, which can be edited in any webpage.

I then edit the navigation buttons so that the user has a better navigation experience in the website.

To do this, I head to Adobe Fireworks and create buttons for navigation.

Making Buttons with Fireworks

These navigation buttons were created in Fireworks using vectors and text. To create them into an interactive button, however, I have to convert the vectors into a symbol.

From there, I tweak the different states of the button (ie. unclicked, hovered, and clicked).

Afterwards, I export the button to Dreamweaver as a Fireworks HTM file.

I then create another AP Div for the Fireworks HTM file, and now the buttons are ready to use in my planning site (as seen in the navigation bar in the left).

 

 

 

Making Buttons with Text

Click on the picture above for an example of a navigation made by text (html/css).

Navigation could be made by using an unordered list, and tweaking it with CSS.

After making an unordered list comprising of the subpages, I created a new CSS Rule to customize the list. I removed the list type style, so that it does not have a bullet point with it, and I changed its alignment/float to the right so that the list is horizontal instead of vertical. I also adjusted the margins so that the navigation is more orderly. I added a hover property as well to the links so whenever you hover the text in the links, it changes color. It also changes color after you visited the link.

Compare and Contrast: Navigation with Text and FW

Text Fireworks
  • Using text (HTML/CSS) to create navigation is considered more efficient as it's more fast.
  • It's less aesthetic than making buttons from Fireworks.
  • Making text navigation requires CSS, which can be time consuming.
  • More simplistic which can serve some website's purposes
  • More graphic based, which means it is easier to manipulate the look according to the way we want it to look like.
  • The way of making buttons using fireworks is more click-based, which doensn't require as much typing compared to HTML/CSS.
  • It is very easy to change the states of the buttons, like the hovering state, or post-click state. This is because Fireworks has the feature to create graphical states for buttons.

Verdict

In the end I used navigation buttons made from Fireworks because I feel like it's much more easy to adjust graphically and because I want my site to look a bit more aesthetic with the least effort, Fireworks was ideal to use. I do enjoy making text navigations but the states made from typing HTML/CSS can be quite tedious and time-consuming and I'd rather take a shorter route.
